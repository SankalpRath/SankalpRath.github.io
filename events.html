Internet Mystery

Internet Mystery

This is an event to test the deciphering abilities of participants. Participants will need to solve some mind-boggling puzzles to get to the treasure. This event will be conducted in two phases. Only the best one will win the prize

Specification:

Phase 1- it will be a pre-event. Participants will need to solve all the puzzles to know when and where phase 2 will be conducted

Phase 2- the final puzzle will be provided on the spot. The very first person to solve the puzzle will win the prize.

Rules:

  • Number of participants-1
  • Full use of internet is allowed.
Suspense Coding

Suspense Coding

This is an event to test the algorithmic understanding of participants. Participants will need to write simple programs for which they will receive points. The suspense in this event is the language to be used which will be revealed on the spot.

Specification:

Participants will need to write simple programs like addition of numbers, finding factorial etc. with an unknown language. They will have full access to the documentation of the language. One to store the highest point will win.

Rules:

  • Participants per group- maximum 2
  • Full access to internet.
  • Time- 1 hr.
Hour of code

Hour of code

In this event participant’s knowledge, skills and accuracy in coding will be tested. The participants are free to use any programming language they are comfortable with.

Specification:

They will be some questions for which participants have to write the code. The judgement will be based on the progress and performance of code. Step marking is allowed and the one with highest points will win.

Rules:

  • No of pariticipants-1
  • Time:1 hr
  • Use of internet is prohibited.
Fast and Curious

Fast and Curious

This event is to test programmers ability to improve the performance of a code. Points will be rewarded on the basis of what they change in the program and how much it reduces the running time of the program.

Specification:

Some pre-written programs will be provided each with a scope of improvement. Participants can make any changes to program unless the output remains logically correct. This event will be an only c-programming event and the compiler used will be gcc.

Rules:

  • Participants per team-maximum 2
  • Only C language.
  • Only gcc compiler.
  • Time: 1 hr.
  • Use of internet is prohibited.
Debugger

Debugger

This is an event to test the error correcting capabilities of the participants. The participants will be given several pieces of code each containing some errors which they need to correct.

Specification:

Participants will be given some pieces of code which may have syntactical error or logical error which the participant needs to correct. Points will be rewarded only if the program has no syntactical error and produces correct result.

Rules:

  • Number of participants- maximum 2 per group.
  • Time: 1 hr.
  • Use of internet is prohibited.
Blind coding

Blind coding

Ever wondered what is to type code without seeing what you are writing. This event will let you experience this. You will have to write code when your monitor is turned off.

Specification:

Participants will be given a question for which they need to write a program in paper. After they are done, they will need to type their program into their computer by turning off the monitor. Points will be rewarded on how many mistakes they made during typing.

Rules:

  • Number of participants- 1 .
  • Time: 1 hr.
  • Monitor should be off while typing.

Techno cupid

Techno cupid

The most awaited couple event from last year is back!. In this a male and a female coder needs to work with each other to defeat other couples. It consists of two rounds each related to programming and partnership. The best pair will win.

Specification:

Round 1- In this round couples will be given a piece if code and they have to type and run it within specified time. The code may have errors which they need to correct. Top 10 pairs with fastest performance will be selected for next round.

Round 2- This round is based on mutual understanding and how much do they know about each other.

Rules:

  • No of participants-2(one male, one female).
  • Time:1hr *2(for 2 rounds)
  • Use of internet is prohibited.
